- **계정**: nexokorea@gmail.com

### 현재 설정
- **Cloud Name**: `dthtfs1mf`
- **Upload Preset**: `nexo-reviews-unsigned`
- **저장 폴더**: `nexo-reviews`

### 무료 플랜 제한
- **저장 용량**: 월 25GB
- **대역폭**: 월 25GB
- **변환**: 월 25,000건
- **관리 API**: 월 5,000건

### 작동 방식

#### 1. 이미지 업로드 프로세스
1. 관리자가 후기 추가 시 이미지 선택
2. 이미지를 FormData로 변환
3. Cloudinary API로 업로드
4. Cloudinary에서 URL 반환 (예: `https://res.cloudinary.com/dthtfs1mf/image/upload/v1234567890/nexo-reviews/abc123.jpg`)
5. URL을 localStorage에 저장
6. 모든 사용자가 동일한 이미지 URL로 접근

#### 2. 코드 위치
- **설정**: `index.html` 1273-1274줄
- **업로드 함수**: `index.html` 1458줄 (`uploadImageToCloudinary`)
- **사용 위치**: `index.html` 1537줄 (후기 추가 시)

#### 3. 에러 처리
- Cloudinary 설정이 없거나 업로드 실패 시:
  - 자동으로 Base64로 저장 (기존 방식)
  - 후기는 정상적으로 저장됨
  - 사용자에게 에러 메시지 표시

### 관리 방법

#### 이미지 확인
1. Cloudinary 대시보드 접속: https://console.cloudinary.com
2. **Media Library** 클릭
3. `nexo-reviews` 폴더에서 업로드된 이미지 확인

#### 이미지 삭제
1. Media Library에서 이미지 선택
2. 삭제 버튼 클릭
3. ⚠️ **주의**: 이미지 삭제 시 웹사이트에서도 표시되지 않음

#### 사용량 확인
1. Dashboard → **Usage** 탭
2. 저장 용량, 대역폭, 변환 건수 확인
3. 무료 한도 초과 시 알림

### 설정 변경 방법

#### Cloud Name 변경
```javascript
// index.html 1273줄
const CLOUDINARY_CLOUD_NAME = '새로운-cloud-name';
```

#### Upload Preset 변경
```javascript
// index.html 1274줄
const CLOUDINARY_UPLOAD_PRESET = '새로운-preset-name';
```

#### 새 Upload Preset 생성
1. Cloudinary Dashboard → **Settings** → **Upload**
2. **Upload presets** → **Add upload preset**
3. 설정:
   - **Preset name**: 원하는 이름
   - **Signing mode**: `Unsigned` (필수)
   - **Folder**: `nexo-reviews` (선택)
4. **Save** 클릭

### 보안

#### Upload Preset 보안
- **Unsigned preset** 사용 (서버 없이 업로드 가능)
- Preset 이름은 프론트엔드에 노출됨 (안전함)
- 필요시 **Signed preset** 사용 가능 (더 안전하지만 서버 필요)

#### API Key/Secret
- 현재는 사용하지 않음 (Unsigned preset)
- Signed preset 사용 시 필요

### 문제 해결

#### 이미지가 업로드되지 않음
1. Cloudinary 설정 확인
   - Cloud Name이 올바른지 확인
   - Upload Preset이 존재하는지 확인
   - Upload Preset이 `Unsigned`로 설정되어 있는지 확인

2. 브라우저 콘솔 확인
   - F12 → Console 탭
   - 에러 메시지 확인

3. 네트워크 확인
   - F12 → Network 탭
   - Cloudinary API 요청 확인

#### 이미지가 표시되지 않음
1. 이미지 URL 확인
   - localStorage에서 후기 데이터 확인
   - 이미지 URL이 Cloudinary URL인지 확인

2. Cloudinary에서 이미지 존재 확인
   - Media Library에서 이미지 확인

3. CORS 문제
   - Cloudinary는 CORS를 지원하므로 문제 없음

### 비용 관리

#### 무료 플랜 초과 시
- **저장 용량 초과**: 이미지 업로드 불가
- **대역폭 초과**: 이미지 로드 느려짐
- **해결 방법**:
  1. 오래된 이미지 삭제
  2. 이미지 최적화 (자동 제공)
  3. 유료 플랜 업그레이드 ($89/월부터)

#### 최적화 팁
- Cloudinary가 자동으로 이미지 최적화 제공
- 필요시 이미지 크기 제한 (코드에서)
- 정기적으로 사용하지 않는 이미지 삭제

### API 엔드포인트

#### 업로드 엔드포인트
```
POST https://api.cloudinary.com/v1_1/{cloud_name}/image/upload
```

- `upload_preset`: Upload Preset 이름
- `folder`: 저장 폴더 (선택)

#### 응답 형식
```json
{
  "public_id": "nexo-reviews/abc123",
  "secure_url": "https://res.cloudinary.com/dthtfs1mf/image/upload/v1234567890/nexo-reviews/abc123.jpg",
  "url": "http://res.cloudinary.com/...",
  "width": 1920,
  "height": 1080,
  "format": "jpg",
  "bytes": 245678
}
```

## 🔄 배포 롤백 (Rollback)

### 롤백이란?
문제가 발생했을 때 이전 정상 작동하던 배포 버전으로 되돌리는 기능입니다.

### 롤백 방법

#### 방법 1: Netlify 대시보드에서 롤백
1. Netlify 대시보드 접속: https://app.netlify.com
2. 프로젝트 "nexokorea" 선택
3. **"Deploys"** 탭 클릭
4. 배포 목록에서 롤백할 배포 찾기
5. 해당 배포의 **"..."** 메뉴 클릭
6. **"Publish deploy"** 선택
7. 확인 클릭

**장점**: 간단하고 빠름, 시각적으로 확인 가능

#### 방법 2: Netlify CLI로 롤백
```bash
# 배포 목록 확인
netlify deploy:list

# 최근 배포로 롤백
netlify deploy:rollback

# 특정 배포로 롤백 (배포 ID 필요)
netlify deploy:rollback --to=<deploy-id>
```

**장점**: 터미널에서 빠르게 실행 가능

### 롤백 시 주의사항
- ⚠️ 롤백하면 최신 변경사항이 사라집니다
- ⚠️ 롤백 전에 현재 버전의 문제점을 기록해두세요
- ✅ 롤백 후에도 배포 히스토리는 유지됩니다
- ✅ 언제든지 다시 최신 버전으로 복구 가능합니다

### 배포 히스토리 확인
- **위치**: Netlify 대시보드 → Deploys
- **기능**:
  - 모든 배포 기록 확인
  - 각 배포의 상태 (성공/실패)
  - 배포 시간 및 변경사항
  - 배포별 고유 URL

## 📥 로컬 파일을 이전 배포 시점으로 되돌리기

### 방법 1: Netlify에서 이전 배포 파일 다운로드 (권장)

#### 단계별 가이드
1. **Netlify 대시보드 접속**
   - https://app.netlify.com 접속
   - 프로젝트 "nexokorea" 선택

2. **Deploys 탭으로 이동**
   - 왼쪽 메뉴에서 "Deploys" 클릭
   - 롤백할 배포 찾기

3. **배포 상세 페이지 접속**
   - 해당 배포 클릭하여 상세 페이지로 이동

4. **파일 다운로드**
   - 배포 상세 페이지에서 "Published files" 또는 "Browse files" 클릭
   - 필요한 파일들을 다운로드
   - 또는 배포 URL에서 직접 파일 접근:
     ```
     https://[deploy-id]--nexokorea.netlify.app/index.html
     ```

5. **로컬 파일 교체**
   - 다운로드한 파일로 현재 파일 교체
   - 저장 후 확인

#### 빠른 방법 (터미널)
```bash
# 특정 배포 ID의 파일 다운로드
curl -o index.html https://[deploy-id]--nexokorea.netlify.app/index.html

# 예시: 배포 ID가 6940f4a22bc02bae085d6e06인 경우
curl -o index.html https://6940f4a22bc02bae085d6e06--nexokorea.netlify.app/index.html
```
